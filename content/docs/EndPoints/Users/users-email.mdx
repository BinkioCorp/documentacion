---
title: Email
description: Este endpoint permite obtener información de un usuario específico utilizando su correo electrónico.

---

## GET `/api/users/email/{email}`

### Solicitud

**Método:** GET
**Ruta:** `/api/users/email/{email}`

**Cuerpo de la solicitud:**

```json
{
  "email": "string"
}

```

### Detalle de los Parámetros de consulta

| **Nombre**        | **Tipo**  | **Requerido** | **Descripción**                                                      |
|-------------------|-----------|---------------|----------------------------------------------------------------------|
| `email`           | `string`  | Sí            | Correo electrónico del usuario que se desea consultar.               |


### Respuestas

**200 - Consulta exitosa**

```json
{
  "id": "12345",
  "email": "usuario@dominio.com",
  "name": "Juan Pérez",
  "status": "active",
  "createdAt": "2024-11-27T12:34:56.000Z"
}

```

**404 - Usuario no encontrado**

```json
{
  "message": "No se encontró un usuario con el correo electrónico especificado."
}

```

### Notas

El correo electrónico (**email**) debe ser válido y estar registrado en el sistema.

### Ejemplo de Uso

**Solicitud HTTP:**

```https
GET /api/users/email/usuario@dominio.com HTTP/1.1
Host: api.ejemplo.com
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
Content-Type: application/json

```

**Respuesta Exitosa:**

```json
{
  "id": "12345",
  "email": "usuario@dominio.com",
  "name": "Juan Pérez",
  "status": "active",
  "createdAt": "2024-11-27T12:34:56.000Z"
}

```

**Respuesta con Error:**

```json
{
  "message": "No se encontró un usuario con el correo electrónico especificado."
}

```
